# Contributing

Thanks for your interest 🙌

## Guidelines

- Keep code simple and readable
- Prefer small, focused commits
- Comment non-obvious DSP behavior

## Branch Naming

Use the format <target_branch_name>-<feature_description>-<your_name>

Example:
`pair2-reword_documentation-victorien`

## Commit Messages

Format:

[type] file: short description

For the commit message in itself, spend effort explaining __why__ the change was required rather
than what changed.

## Pull requests

- Describe what changed and why
- Add testing notes if relevant
- Keep PRs small when possible
